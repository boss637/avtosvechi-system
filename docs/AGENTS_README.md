# 🤖 README: Система Агентов «Триада»

## Интеллектуальная система мониторинга и автоматизации

---

## 🎯 ОБЗОР СИСТЕМЫ

**Триада Агентов** — промышленная система автоматического мониторинга,  
реагирования и решения проблем в режиме 24/7.

### Архитектурная схема:

```text
          [ИНЦИДЕНТ]
              │
      ┌───────┼───────┐
      ▼       ▼       ▼
     👁️      🤖     🧠
WATCHER    EXECUTOR   SOLVER
│             │          │
▼             ▼          ▼
[ФИКСАЦИЯ] [УПРАВЛЕНИЕ] [АНАЛИЗ]
│             │          │
     └────────┴────────┘
              ▼
          [РЕШЕНИЕ]
```

## 🏗️ АРХИТЕКТУРА АГЕНТОВ

### 1. 👁️ **WATCHER AGENT** — Агент Наблюдения
**Назначение:** Мониторинг системы в реальном времени  
**Порт:** 9090  
**Директория:** `watcher-agent/`

**Функции:**
- 📊 Отслеживание метрик системы
- 🔍 Обнаружение аномалий
- 📝 Сбор логов и диагностика
- 🚨 Генерация алертов

**Технологии:** Prometheus, Grafana, custom metrics

### 2. 🤖 **EXECUTOR AGENT** — Агент Исполнитель
**Назначение:** Автоматическое выполнение действий  
**Порт:** 8000  
**Директория:** `executor-agent/`

**Функции:**
- 💬 Обработка команд из Telegram
- ⚡ Выполнение скриптов
- 🎛️ Управление ресурсами
- 🔄 Автоматическое восстановление

**Технологии:** Python, Telegram Bot API, Docker API

### 3. 🧠 **SOLVER AGENT** — Агент Решатель
**Назначение:** Анализ и решение сложных проблем  
**Порт:** 8080  
**Директория:** `solver-agent/`

**Функции:**
- 🧩 Анализ корреляций событий
- 💡 Предложение решений
- 📈 Прогнозирование проблем
- 🎯 Оптимизация работы системы

**Технологии:** Machine Learning, правила, алгоритмы

## 🚀 БЫСТРЫЙ СТАРТ

### Установка и запуск:
```bash
# Клонирование репозитория
git clone https://github.com/boss637/avtosvechi-system.git
cd avtosvechi-system

# Запуск всех агентов
docker-compose up -d watcher executor solver

# Проверка статуса
docker-compose ps
```

### Проверка работы:
```bash
# Watcher (метрики)
curl http://localhost:9090/metrics

# Executor (здоровье)
curl http://localhost:8000/health

# Solver (статус)
curl http://localhost:8080/status
```

# 🔧 КОНФИГУРАЦИЯ

##Основные файлы конфигурации**:

1. **.env — переменные окружения**

2. **docker-compose.yml — Docker конфигурация**

3. **watcher-agent/config.yaml — конфиг Watcher**

4. **executor-agent/config.py — конфиг Executor**

5. **solver-agent/rules.yaml — правила Solver**
